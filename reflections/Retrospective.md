# Retrospective

## Things we would do differently

1. **Adding dates to the project plan** — we assigned tasks to people but didn't set target dates, which made it harder to track progress against the deadline.
2. **Agreeing on file structure earlier** — the folder structure and naming conventions (e.g. how resources are linked) weren't agreed upfront, leading to some inconsistency that needed fixing later.
3. **Clearer formatting guidelines from the start** — each section ended up with slightly different styles before being standardised. Agreeing on a template early would have saved time.
4. **More frequent communication** — Calling more often to discuss how we were doing would have helped get work done faster.
5. **Branching off the wrong branch** — being more careful about which branch you're on before starting new work would avoid having to undo commits.
6. **Contribution imbalance risk** — with three people working across many tasks it wasn't always clear who was blocked waiting for someone else to finish. A simple shared tracker or daily update in a group chat would have helped.
7. **Dependencies between tasks weren't made explicit** — for example, the handbook content couldn't really be written until the resources were found, but this wasn't noted in the project plan. Being clearer about task dependencies upfront would help with scheduling.
8. **Spelling and grammar checks before merging** — a typo in the project plan made it through the review process and needed a separate branch and PR just to fix one word. Adding a checklist to PRs could prevent this.
